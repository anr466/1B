## القاعدة العليا

```
لا تفهم = لا تخطط
لا تخطط = لا تنفذ
```

## التسلسل الإلزامي

```
[اكتشاف] → [استكشاف] → [تحليل] → [خطة] → [موافقة] → [تنفيذ] → [تحقق] → [توثيق]
    ↑________↓_____________↓
    (هذا النصف أهم من النصف الثاني)
```

---

## 1. اكتشاف (Discover) — إلزامي

**قبل أي شيء:**
```bash
git log --oneline -10
ls -la
find . -type f -name "*.py" -o -name "*.ts" -o -name "*.dart" | head -30
```

**اسأل نفسك:**
- ما نوع هذا المشروع؟
- ما التقنيات المستخدمة؟
- ما البنية العامة؟

**لا تتجاوز هذه الخطوة.**

---

## 2. استكشاف (Explore) — إلزامي

**لكل ملف يتعلق بالأمر:**
```
□ من يستدعي هذا الملف؟ (ابحث في كل المشروع)
□ ما الملفات التي يستدعيها؟
□ ما البيانات الداخلة؟ من أين تأتي؟
□ ما البيانات الخارجة؟ إلى أين تذهب؟
□ ما الحالة قبله؟ ما الحالة بعده؟
```

**ارسم التدفق:**
```
[أ] → [ب] → [ج] → [د]
 ↑________________↓
```

**لا تتجاوز هذه الخطوة حتى ترسم التدفق.**

---

## 3. تحليل (Analyze) — إلزامي

**لكل نقطة في التدفق:**
```
□ ما يحدث هنا؟
□ لماذا يحدث هنا؟
□ ماذا يحدث إذا فشل؟
□ من يتأثر؟
```

**حدد:**
- نقطة البداية (Entry Point)
- نقطة النهاية (Exit Point)
- التسلسل (Sequence)
- التفرعات (Branches)
- التكاملات (Integrations)

**لا تتجاوز هذه الخطوة حتى تفهم كل نقطة.**

---

## 4. خطة (Plan) — بعد الفهم الكامل

**بناءً على الاستكشاف والتحليل:**
```
المهمة 1: [ملف] ← [تغيير]
  التدفق: [أ] → [ب] → [ج]
  يؤثر على: [د], [هـ]
  يعتمد على: [و], [ز]
  اختبار: [أمر محدد]

المهمة 2: [ملف] ← [تغيير]
  التدفق: [ج] → [ح] → [ط]
  ...
```

**اعرض للمستخدم. انتظر "نعم".**

---

## 5. موافقة (Approve) — إلزامي

**المستخدم يقول "نعم" → انتقل**
**المستخدم يسأل → أجب من الاستكشاف**
**المستخدم يعدل → عدّل واعرض مجدداً**

---

## 6. تنفيذ (Execute)

**لكل مهمة:**
```
□ نفّذ التغيير
□ اختبر التدفق الكامل (ليس فقط النقطة)
□ نجح → التالية
□ فشل → أصلح مرة → اختبر التدفق الكامل → نجح أو أبلغ
```

---

## 7. تحقق (Verify) — التدفق الكامل

```
□ من البداية إلى النهاية
□ كل التكاملات
□ كل التفرعات
□ لا شيء انكسر
```

---

## 8. توثيق (Document)

```bash
git add -A
git commit -m "[نوع]: [ما أُنجز] | flow: [التدفق] | next: [التالي]"
git push
```

## مثال: تغيير كلمة المرور

```
[اكتشاف]
المشروع: تطبيق تداول Flutter + FastAPI
البنية: Mobile → API → DB

[استكشاف]
الملف: auth_service.py
  من يستدعيه؟: auth_controller.py, user_settings.dart
  يستدعي: db_users_mixin.py, email_service.py
  داخلة: old_password, new_password, user_id
  خارجة: success/failure, email_notification

التدفق:
[UI: إعدادات] → [API: /change-password] → [Service: تحقق]
    → [DB: تحديث] → [Email: إشعار] → [UI: رسالة نجاح]
              ↓
        [إذا فشل: رسالة خطأ]

[تحليل]
نقطة البداية: UI - زر "تغيير"
نقطة النهاية: UI - رسالة نجاح/فشل
التسلسل: 6 خطوات
التكاملات: DB, Email, UI
نقاط الضعف: DB قد تفشل, Email قد يتأخر

[خطة]
"سأعدّل:
 1. auth_service.py ← تحقق أقوى → اختبر: تدفق كامل
 2. db_users_mixin.py ← معاملة آمنة → اختبر: rollback يعمل
 3. email_service.py ← إشعار تلقائي → اختبر: يُرسل
 تنفيذ؟"

[موافقة]
المستخدم: "نعم"

[تنفيذ]
✅ 1/3 auth_service.py (اختبر: تدفق كامل نجح)
✅ 2/3 db_users_mixin.py (اختبر: rollback نجح)
✅ 3/3 email_service.py (اختبر: إشعار وصل)

[تحقق]
التدفق الكامل: من UI إلى Email → نجح
التكاملات: كلها تعمل
لا انكسار

[توثيق]
git commit -m "fix: password change | flow: ui→api→db→email→ui"
أبلغ: "✅ تم. التدفق الكامل يعمل."
```

## قواعد صارمة

| القاعدة | الجزاء |
|---------|--------|
| لا اكتشاف = لا استكشاف | توقف |
| لا استكشاف = لا تحليل | توقف |
| لا تحليل = لا خطة | توقف |
| لا خطة = لا تنفيذ | توقف |

---

## ما يجب فهمه قبل القرار

```
قبل أن تقول "سأفعل":
□ فهمت المشروع؟
□ فهمت الملف؟
□ فهمت من يستدعيه؟
□ فهمت من يستدعي؟
□ فهمت التدفق؟
□ فهمت التكاملات؟
□ فهمت نقاط الضعف؟

إذا لا → استكشف أكثر
```

---

## الخلاصة

```
اكتشف → استكشف → حلل → افهم → خطط → نفذ
```

**النصف الأول أهم من النصف الثاني.**
**الفهم العميق = القرار الصحيح.**
```

---

## الفرق الجوهري

| قبل | بعد |
|-----|-----|
| فهم سطحي | **اكتشاف + استكشاف + تحليل إلزامي** |
| خطة بدون سياق | **خطة مبنية على خريطة تدفق كاملة** |
| اختبار نقطة | **اختبار تدفق كامل** |
| "سأفعل" فوراً | **"فهمت، وهذا التدفق" ثم قرار** |

## التسلسل المرئي الجديد

```
    [اكتشاف] ← المشروع ككل
      ↓
    [استكشاف] ← الملف والتدفق
      ↓
    [تحليل] ← كل نقطة وتكامل
      ↓
    [خطة] ← مبنية على الفهم
      ↓
    [موافقة] ← المستخدم
      ↓
    [تنفيذ] ← تدفق كامل
      ↓
    [تحقق] ← تدفق كامل
      ↓
    [توثيق] ← git
```

**النموذج الآن: مستكشف أولاً، منفذ ثانياً.** ✅
