## 🎯 المبادئ الأساسية

### 1️⃣ هوية المستخدم (User Identity)

#### User ID
- **ثابت وغير قابل للتغيير** ✅
- مرتبط بسجل المستخدم في قاعدة البيانات
- يُستخدم لجميع العمليات والصلاحيات

#### Username
- **ثابت وغير قابل للتغيير** ✅
- يُستخدم كمرجع تقني
- يُعرض في الملف الشخصي (للقراءة فقط)

#### الاسم الكامل (Name)
- **قابل للتغيير بدون تحقق** ✅
- يُستخدم للعرض في الواجهة
- حفظ مباشر بدون OTP

## 🔐 طرق تسجيل الدخول

### 1. تسجيل الدخول التقليدي

**التدفق:**
```
1. إدخال البريد/الجوال + كلمة المرور
2. التحقق من البيانات
3. عند النجاح:
   ✅ حفظ بيانات الدخول بشكل آمن (Secure Storage)
   ✅ حفظ JWT Token
   ✅ إنشاء الجلسة
```

**الملفات المسؤولة:**
- `src/screens/LoginScreen.js`
- `src/services/SecureStorageService.js`

---

### 2. تسجيل الدخول بالبصمة (Biometric Login)

**الشروط:**
- ✅ تسجيل دخول سابق ناجح
- ✅ البصمة مُفعلة من الإعدادات
- ✅ "تذكرني" مُفعل
- ✅ كلمة المرور محفوظة (مشفرة)

**التدفق:**
```
1. فتح التطبيق
2. فحص وجود بيانات محفوظة
3. إذا البصمة مفعلة:
   ✅ عرض طلب البصمة
   ✅ عند النجاح → دخول تلقائي
   ✅ عند الفشل → عرض شاشة تسجيل الدخول
```

**الملفات المسؤولة:**
- `src/services/BiometricService.js`
- `App.js` (فحص أولي)
- `src/screens/LoginScreen.js` (احتياطي)

## ✅ آلية التحقق الموحدة (Unified Verification)

### طريقة التحقق الافتراضية

**SMS (رمز تحقق OTP)**
- الطريقة الأساسية للتحقق
- يُرسل عبر Firebase Auth

**Email OTP (خيار بديل)**
- متاح كخيار ثانوي
- يُستخدم عند عدم توفر SMS

---

### العمليات التي تتطلب تحقق

| العملية | طريقة التحقق | ملاحظات |
|---------|--------------|----------|
| **تغيير كلمة المرور** | SMS/Email | يتطلب كلمة المرور القديمة |
| **تغيير الإيميل** | SMS فقط | التحقق من الجوال |
| **تغيير رقم الجوال** | Email فقط | التحقق من الإيميل |
| **تفعيل/إلغاء البصمة** | SMS/Email | تأكيد الهوية |
| **تغيير مفاتيح Binance** | SMS/Email | عملية حساسة |
| **حذف مفاتيح Binance** | SMS/Email | عملية حساسة |

**لا تتطلب تحقق:**
- ✅ تغيير الاسم الكامل (Name) - حفظ مباشر

### التغييرات الحساسة


## ⚠️ ملاحظات مهمة

1. **عدم التكرار:**
   - لا تطلب OTP مرتين للعملية الواحدة
   - استخدم `pendingAction` للتتبع

2. **الأمان:**
   - جميع البيانات الحساسة مشفرة
   - JWT Token يُحدث تلقائياً
   - Session timeout = 1 ساعة

3. **تجربة المستخدم:**
   - رسائل خطأ واضحة بالعربي
   - إمكانية إعادة المحاولة
   - تأكيد للعمليات الحساسة

4. **التوافق:**
   - يعمل على iOS و Android
