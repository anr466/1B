# 📊 دليل عرض البيانات - نظام التداول الآلي

**التاريخ:** 10 يناير 2026  
**المبدأ الأساسي:** عرض ما يحتاجه المستخدم للمراقبة فقط، بدون تكرار

---

## ⚡ طبيعة النظام

**نظام تداول آلي 100%**

```
المستخدم → يراقب النتائج فقط
النظام الخلفي → يتداول تلقائياً
```

### ❌ المستخدم لا يحتاج

- أدوات التداول اليدوي
- أزرار "شراء" أو "بيع"
- تفاصيل تقنية معقدة
- معلومات متكررة
- بيانات غير قابلة للتطبيق

### ✅ المستخدم يحتاج

- **الرصيد الحالي** - كم لديّ؟
- **الأرباح/الخسائر** - كيف أداء النظام؟
- **الصفقات النشطة** - ماذا يحدث الآن؟
- **السجل** - ماذا حدث سابقاً؟
- **الإعدادات** - كيف يتداول النظام؟

## 🎯 البيانات الأساسية (لا تكرار)

### 1. الرصيد والأداء

**يُعرض مرة واحدة فقط - في Dashboard**

```javascript
<BalanceCard
    balance={10250.75}           // الرصيد الإجمالي
    todayChange={125.50}         // التغيير اليومي
    todayChangePercent={1.24}    // النسبة اليومية
    totalProfit={1250.75}        // الربح الكلي
    totalProfitPercent={13.89}   // النسبة الكلية
/>
```

**❌ لا يُعرض:**
- في Portfolio Screen (تكرار)
- في Settings Screen (غير ذي صلة)

**✅ مكان العرض:** Dashboard فقط

---

### 2. الإحصائيات

**يُعرض مرة واحدة - في Dashboard**

```javascript
<View style={styles.statsGrid}>
    <StatCard
        title="عدد الصفقات"
        value={45}
        icon="activity"
    />
    <StatCard
        title="معدل النجاح"
        value="84%"
        icon="target"
    />
    <StatCard
        title="المراكز النشطة"
        value={3}
        icon="trending-up"
    />
</View>
```

**❌ لا يُعرض:**
- تفاصيل كل صفقة في Dashboard (زحمة)
- إحصائيات مكررة في شاشات أخرى

**✅ مكان العرض:** Dashboard - ملخص سريع فقط

---

### 3. المراكز النشطة (الصفقات المفتوحة)

**يُعرض في مكانين مختلفين:**

#### Dashboard - ملخص
```javascript
<ActivePositionsCard
    positions={activePositions}
    compact={true}              // عرض مضغوط
    limit={3}                   // أقصى 3 صفقات
    onViewAll={() => navigate('Portfolio')}
/>
```

**يعرض:** رمز العملة، الربح/الخسارة الحالي فقط

#### Portfolio - تفصيلي
```javascript
<FlatList
    data={activePositions}
    renderItem={({ item }) => (
        <TradeCard
            trade={item}
            compact={false}     // عرض كامل
        />
    )}
/>
```

**يعرض:** جميع التفاصيل (السعر، الكمية، إلخ)

**✅ لا تكرار:** كل شاشة لها غرض مختلف

---

### 4. المحفظة (الأصول)

**يُعرض في Portfolio Screen فقط**

```javascript
<FlatList
    data={assets}
    renderItem={({ item }) => (
        <AssetCard
            asset={{
                symbol: item.symbol,
                amount: item.amount,
                value_usd: item.value_usd,
                profit_loss: item.profit_loss,
                // بدون تكرار الرصيد الإجمالي
            }}
        />
    )}
/>
```

**❌ لا يُعرض:**
- الرصيد الإجمالي (موجود في Dashboard)
- الأرباح الكلية (موجودة في Dashboard)

**✅ يُعرض:** الأصول الفردية فقط

---

### 5. سجل الصفقات

## 🚫 البيانات التي لا يجب عرضها

### 1. تفاصيل تقنية غير ضرورية

```
❌ معرف الصفقة (Trade ID)
❌ رمز الاستراتيجية الداخلي
❌ حالات التنفيذ التقنية
❌ أخطاء النظام التقنية (للمطورين فقط)
```

**السبب:** المستخدم لا يحتاجها للمراقبة

---

### 2. بيانات مكررة

```
❌ الرصيد في كل شاشة
❌ نفس الإحصائيات في أكثر من مكان
❌ نفس المراكز النشطة في Dashboard و Portfolio بنفس التفصيل
```

**السبب:** تشوش المستخدم وتسبب ارتباك

---

### 3. أدوات التداول اليدوي

```
❌ زر "شراء الآن"
❌ زر "بيع الآن"
❌ حقول إدخال السعر/الكمية يدوياً
❌ أوامر سوقية يدوية
```

**السبب:** النظام آلي 100%، المستخدم يراقب فقط

---

### 4. معلومات غير قابلة للتطبيق

```
❌ "سعر الطلب" للمستخدم (النظام يقرر)
❌ "استراتيجية مخصصة" (النظام يختار تلقائياً)
❌ "إغلاق يدوي" (النظام يغلق حسب الإعدادات)
```

**السبب:** المستخدم لا يتحكم، النظام يعمل آلياً

## ✅ قواعد عرض البيانات

### القاعدة 1: مرة واحدة فقط
```
كل معلومة تُعرض في مكان واحد مناسب
مثال: الرصيد → Dashboard فقط
```

### القاعدة 2: السياق المناسب
```
كل بيانة في الشاشة المناسبة لها
مثال: الإعدادات → TradingSettings فقط
```

### القاعدة 3: لا تفاصيل زائدة
```
عرض ما يحتاجه المستخدم للمراقبة فقط
مثال: الربح/الخسارة → نعم، معرف الصفقة → لا
```

### القاعدة 4: ملخص في Dashboard، تفصيل في الشاشة المخصصة
```
Dashboard → آخر 3 مراكز نشطة (ملخص)
Portfolio → جميع المراكز النشطة (تفصيل)
```
